# DGentic 0.2.0 Release Notes

Release date: 2026-05-07

## Summary

DGentic 0.2.0 expands the backend MVP from planning-only foundations into persisted sprint state, guarded filesystem operations, guarded CLI execution, local provider discovery, and scored routing contracts.

## Included

- Local JSON persistence for:
  - Task plans
  - Task runs
  - Event logs
  - Agent briefs
  - Memory records
  - Tool manifests
  - Session summaries
- Task history endpoints:
  - `GET /tasks/plans`
  - `GET /tasks/runs`
- Guarded UTF-8 text file operations:
  - `POST /filesystem/read`
  - `POST /filesystem/write`
- Guarded CLI execution:
  - `POST /cli/execute`
- Local provider probes:
  - Ollama `/api/tags`
  - LM Studio `/v1/models`
- Scored routing decisions with candidate scores.
- Updated documentation and sprint progress logs.

## Configuration

New settings:

- `DGENTIC_DATA_DIR=.dgentic`
- `DGENTIC_OLLAMA_BASE_URL=http://127.0.0.1:11434`
- `DGENTIC_LM_STUDIO_BASE_URL=http://127.0.0.1:1234`

## Known Limitations

- Ollama and LM Studio generation calls are not implemented yet.
- External provider adapters are still placeholders.
- CLI approval is represented by an API field, not a user-facing approval queue.
- Filesystem runtime support is limited to UTF-8 text reads and writes.
- Tool manifests can be registered, but tools are not executed in a sandbox yet.
- Web UI and VS Code extension work have not started yet.
