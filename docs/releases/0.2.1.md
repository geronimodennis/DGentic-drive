# DGentic 0.2.1 Release Notes

Release date: 2026-05-07

## Summary

DGentic 0.2.1 adds persisted CLI command policy rules so operators can configure command allow, approval, and block behavior without code changes.

## Included

- Persisted CLI command policy rules stored in `cli-command-policy-rules.json`.
- New policy rule match types:
  - `executable`
  - `exact`
  - `contains`
  - `argument_contains`
- Rule priority and enabled/disabled state.
- Matched rule metadata in command policy decisions.
- New endpoints:
  - `POST /cli/policy/rules`
  - `GET /cli/policy/rules`
  - `PATCH /cli/policy/rules/{rule_id}`
- CLI runtime integration so configured rules affect guardrail checks, approval creation, and command execution.
- Agentic workflow documentation under `docs/agentic-workflows/`.

## Known Limitations

- CLI cancellation and streaming/polling output APIs are not implemented yet.
- CLI permissions are not yet scoped by triggering agent, role, or task context.
- Command environment variables still need stricter control and audit contracts.
- External provider adapters are still placeholders.
- Web UI and VS Code extension work have not started yet.
