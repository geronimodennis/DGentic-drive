# DGentic 0.2.2 Release Notes

Release date: 2026-05-07

## Summary

DGentic 0.2.2 adds asynchronous CLI command runs with polling and process-local cancellation, while hardening default command policy against shell-wrapper bypasses.

## Included

- New command run statuses:
  - `running`
  - `completed`
  - `timed_out`
  - `cancelled`
- New endpoints:
  - `POST /cli/runs`
  - `GET /cli/runs/{run_id}`
  - `POST /cli/runs/{run_id}/cancel`
- Persisted asynchronous command run records before process execution.
- Process-local cancellation for currently running commands.
- Shell-wrapper policy inspection for common command wrappers such as:
  - `cmd /c`
  - `sh -c`
  - `bash -c`
  - PowerShell `-Command` / `-c`
- Tests for async completion, polling, cancellation, API cancellation, and shell-wrapped blocked command detection.

## Known Limitations

- Cancellation is process-local; commands running before a backend restart cannot be cancelled after restart.
- Streaming command output is not implemented yet.
- CLI permissions are not yet scoped by triggering agent, role, or task context.
- Command environment variables still need stricter control and audit contracts.
- External provider adapters are still placeholders.
- Web UI and VS Code extension work have not started yet.
