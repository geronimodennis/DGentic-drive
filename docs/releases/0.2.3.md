# DGentic 0.2.3 Release Notes

Release date: 2026-05-07

## Summary

DGentic 0.2.3 releases the autonomous-agent role boundary governance update. It clarifies ownership rules so Developer, QA, Reviewer, Security, PM, and Release Manager agents do not silently cross file ownership boundaries during autonomous workflows.

## Included

- Added role boundary governance:
  - `docs/agentic-workflows/governance/role-boundaries.md`
- Developer Agent constraints:
  - May modify production implementation and implementation docs.
  - Must not create or modify tests, fixtures, snapshots, or QA validation scripts.
- QA Agent constraints:
  - May create and modify tests, fixtures, snapshots, and test-only validation scripts.
  - Must not create or modify production source, runtime implementation, schemas, API implementation, or production configuration.
- Cross-role handoff rules when implementation changes or test changes are needed.
- Updated autonomous mode instructions to require role-boundary enforcement.
- Updated the agent response format with `Write Scope Used`.
- Updated sprint lifecycle and coordination rules to preserve role accountability.

## Known Limitations

- Role boundaries are currently documented governance rules.
- Backend agent orchestration does not yet enforce role write scopes with machine-readable policy.
- GitHub Release asset upload requires GitHub CLI or a GitHub token in the execution environment.
