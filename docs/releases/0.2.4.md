# DGentic 0.2.4 Release Notes

Release date: 2026-05-07

## Summary

DGentic 0.2.4 advances Story 5.3 by adding agent/context-aware CLI permission decisions and controlled command environment handling. Command policy rules can now be scoped to agent roles, command execution results and run history carry agent/task metadata, and explicit environment overrides are validated and audited by key without storing values in approval records.

## Included

- Added command context fields:
  - `agent_role`
  - `agent_id`
  - `task_id`
  - `requested_by`
- Added `agent_roles` scoping on persisted CLI policy rules.
- Added controlled command environment construction with a small inherited baseline.
- Blocked sensitive environment overrides including `PATH`, `PYTHONPATH`, `SYSTEMROOT`, `COMSPEC`, `PATHEXT`, `PYTHONHOME`, `HOME`, and `VIRTUAL_ENV`.
- Added `environment_keys` to command execution results, command run history, approvals, and CLI event metadata.
- Rejected approval-queue requests that include environment values so runtime secrets are not persisted while waiting for approval.
- Added API `400` responses for invalid command environment requests.
- Updated README, architecture documentation, usage guide, developer setup guide, Agile task plan, and progress log.

## Known Limitations

- Streaming command output is not implemented yet.
- Restart-resilient process supervision and stale-running reconciliation are not implemented yet.
- Broader Windows/POSIX parsing validation is still required before production CLI autonomy.
- GitHub Release asset upload requires GitHub CLI or a GitHub token in the execution environment.
