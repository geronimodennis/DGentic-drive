# DGentic 0.2.6 Release Notes

Release date: 2026-05-08

## Summary

DGentic 0.2.6 closes Sprint 8 with the production security and persistence foundation. This release adds fail-closed startup validation when authentication is enabled without usable bearer tokens, expands SQLAlchemy persistence operations with local SQLite backup/restore helpers, and updates sprint/backlog/status documentation.

## Included

- Added authentication configuration validation so staging/production auth cannot silently start without configured bearer tokens.
- Added file-backed SQLite database path resolution, backup, and restore helpers for local/operator smoke workflows.
- Added auth tests for fail-closed configuration behavior.
- Added database tests for SQLite backup/restore round trip and non-SQLite backup rejection.
- Closed Sprint 8 in the refined backlog and progress log, with remaining hardening moved to follow-up backlog.
- Updated README, developer setup, usage, architecture, release distribution, and documentation index entries.
- Bumped package, API, backend `__version__`, and generated tool default version metadata to `0.2.6`.

## Known Limitations

- GitHub Release asset upload requires GitHub CLI, a GitHub token, or the GitHub plugin in the execution environment.
- SQLite backup/restore helpers are intended for local/operator smoke workflows; scheduled remote backup automation and PostgreSQL-native backup operations remain follow-up persistence work.
- Persisted identity, token rotation, bound approval identities, encrypted credential storage, and full audit actor propagation remain future security work.
